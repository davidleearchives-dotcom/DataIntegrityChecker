"혈액수급관리시스템(BMS)의 혈액출고 데이터 정합성 검증 도구(Medical Data Verification Tool)"를 위한 깨끗하고 전문적인 기업용(Enterprise) SaaS 대시보드를 디자인해줘. 핵심 목적은 두 개의 엑셀 파일을 비교하는 거야.

## 레이아웃 요구사항:
1. 사이드바 내비게이션: 대시보드(Dashboard), 검증 이력(History), 설정(Settings) 메뉴 포함.

2. 메인 업로드 섹션: '원본: BMS API 데이터(Source: BMS API Data)'와 '대조본: 적십자사 데이터(Target: Red Cross Data)'라는 라벨이 붙은 두 개의 명확한 업로드 영역(드롭 존).

3. 요약 카드 섹션: 총 건수(Total Rows), 일치(Matched), 불일치(Mismatched), 원본 누락(Missing in Source) 등 4가지 주요 지표를 시각화.

4. 데이터 미리보기 테이블: 요약 섹션 하단에 위치하며, 데이터가 서로 일치하지 않는 행은 연한 빨간색(Light Red)으로 하이라이트 처리.

5. 결과 내보내기 버튼: 우측 상단 혹은 눈에 잘 띄는 위치에 'Export Result (Excel)' 버튼 배치. 그 옆에 로그아웃버튼 표시


## 스타일 가이드:
- 신뢰감을 주는 의료 및 정부 기관 테마를 적용하고, 블루(Blue)와 그레이(Gray) 톤을 주로 사용해줘.
- 군더더기 없는 미니멀하고 기능적인(Minimalist and functional) 디자인이어야 함."

## 메뉴별 상세 화면 및 기능 정의
1. 대시보드 (Dashboard) - 핵심 작업 공간으로, 이 화면은 사용자가 가장 많이 머무르는 곳으로, 실제 데이터 비교와 결과 확인이 이루어지는 메인 워크스페이스입니다.
1) 주요 기능:
- 파일 업로드 영역: 두 개의 독립된 드롭존(BMS 데이터 vs 적십자사 데이터). 파일명, 용량, 업로드 상태 표시.
- 실시간 분석 엔진: '비교 시작' 버튼 클릭 시 두 파일의 칼럼들(설정에서 원본과 대조본 엑셀파일에서 비교할 칼럼들)을 조인하여 대조 수행.
- 검증 결과 요약 (Metric Cards): * 전체 데이터 건수, 일치 건수, 불일치 건수, 누락 데이터(BMS에만 있음/적십자사에만 있음) 수치를 카드로 시각화.
- 데이터 미리보기 (Preview Table):
- 불일치 항목만 필터링해서 보여주는 기능.
- 원본 데이터와 대조본 데이터를 나란히 보여주어 어떤 값이 다른지 즉각 확인.
- 결과 내보내기: 하이라이트 처리가 완료된 엑셀 파일 다운로드.
- 리셋 버튼 : 처음부터 다시하기

2. 검증 이력 (History) - 데이터 트래킹 및 감사, 과거에 수행했던 모든 검증 작업의 기록을 관리하는 화면입니다. 중복 검증을 방지하고 데이터 품질 변화를 모니터링할 때 유용합니다.
1) 주요 기능:
- 이력 리스트: 검증 일시, 원본파일명과 건수, 대조본 파일명과 건수, 일치건수, 불일치 건수, 결과물 재다운로드 내역들을 타임라인 순으로 나열.
- 상세 보기: 특정 이력을 클릭하면 당시 대시보드에서 봤던 요약 수치와 미리보기 데이터를 다시 로드.
- 재다운로드: 과거에 생성했던 '하이라이트 처리된 결과 엑셀 파일'을 언제든 다시 다운로드 가능.
- 검색 및 필터: 검증이력 내역들에 대헛 기간별, 파일명으로 이력을 조회

3. 설정 (Settings) - 시스템 구성 및 관리, 도구의 작동 방식과 사용자 권한을 관리하는 화면입니다.
1) 주요 기능:
- 비교를 위한 매핑 규칙 설정 (Column Mapping): 원본과 대조본 엑셀 파일의 헤더명, 데이터 위치가 조금씩 다를 수 있으므로(예: '병원코드' vs '기관코드'), 이를 칼럼명기준으로 서로 매핑하는 규칙 관리(예. 원본파일에서 비교할칼럼선택: A,B,C,D,E | 대조본파일에서 비교할칼럼선택: F,G,H,I,J,K 칼럼까지 비교), default 비교칼럼은 원본파일에서 A,B,C,D,E 칼럼, 대조본파일에서 A,B,C,D,E 칼럼 칼럼으로 설정되어있는데, 사용자가 로그인후 칼럼을 변경할 수 있음.
- 사용자 관리: 시스템을 이용하는 담당자의 계정 정보, 권한(사용자와 관리자) 및 권한 설정. 추가,수정,삭제기능. 사용자는 자신정보만 확인 및 수정만 가능, 관리자는 관리자본인정보확인 및  수정가능하고, 전체 계정 사용자들 정보를 수정,삭제도 하고 신규 사용자에 대해서 추가도 할 수 있다. 추가시  사용자 아이디, 초기 비밀번호(해당 사용자가 최초로그인시 직접 변경), 이름, 부서명, 연락처정보를 입력할 수 있도록 한다.  아이디는 영문+숫자 4자리 이상, 비밀번호는 영문,숫자,특수문자 조합하여 8자리이상, 이름은 한글 또는 영문글자 10자이내, 부서명은 15자이내, 연락처정보는 숫자만 12자리 이내야. 

4. 로그인
- 아이디 및 비밀번호 입력(비밀번호는 눈표시하여 on/off에 따라 비밀번호 표시가 on/off되도록 해줘)
- default 관리자 id는 admin 이고 비밀번호는 !admin12345 이며, 관리자는 로그인후 설정에서 관리자정보 선택후 자신정보 확인후 비밀번호 변경할 수 있다. 

5. 로그아웃
- 로그인한 사용자가 로그아웃 버튼을 클릭하면 로그인 화면으로 이동.


## 데이터 전처리 및 매핑 규칙:**
- 필수 비교 : 설정 메뉴에서 설정한 칼럼 4개
- Pandas를 사용하여 두 엑셀 데이터를 로드하고, 설정에서 비교칼럼선택한 것을 기준으로 2개의 엑셀 데이터셋을 비교해줘.

**2. 핵심 비교 알고리즘:**
- 두 시트를 Outer Join 하여 다음 케이스를 구분해줘:
  - 'Match': 설정에서 설정한  칼럼들의 데이터가 모두 동일한 경우.
  - 'Mismatch': 설정에서 설정한 칼럼들의 데이터가 하나라도 다른 경우.
  - 'Missing_in_B': 원본(A)에는 있지만 대조본(B)에 없는 데이터.
  - 'Missing_in_A': 대조본(B)에는 있지만 원본(A)에 없는 데이터.

**3. 결과 파일 생성 (openpyxl 사용):**
- 원본 파일(A)의 형태를 유지한 결과 엑셀 파일을 생성해줘.
- 대조본(B)과 비교하여 값이 다른 셀(Cell)은 배경색을 'Yellow'로 해서 원본파일(A)에 칠해야 해.
- 대조본(B)과 비교하여 누락된 열 데이터는 원본파일(A)에 배경색을 'Red'로 해서 생성한다음'누락'으로 표시해줘.
- 마지막 칼럼에 '검증결과'라는 열을 추가하여 '일치', '불일치(필드명)', '누락' 등의 코멘트를 달아줘.

이 로직을 Python `pandas`와 `openpyxl`, `FastAPI`를 사용하는 구조로 작성해줘.
